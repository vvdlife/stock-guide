name: 모듈 리팩토링 (Refactor Module)
description: 특정 파일이나 모듈의 코드 품질을 개선하기 위해 체계적인 리팩토링 절차를 수행합니다.

steps:
  1. **정적 분석 (Analyze)**
     - 리팩토링 대상 코드를 읽고 '코드 냄새(Code Smells)'를 찾아냅니다.
     - 주요 체크 포인트:
       - 너무 긴 함수 (20줄 이상 권장 분리)
       - 높은 인지 복잡도 (중첩된 if/for문)
       - 하드코딩된 값 (매직 넘버/스트링)
       - SOLID 원칙 위배 사항

  2. **개선 계획 수립 (Plan)**
     - 코드를 수정하기 전에 어떻게 변경할 것인지 계획을 세웁니다.
     - 예: "거대한 `ProcessData` 함수를 `Validate`, `Transform`, `Save` 세 개의 작은 함수로 분리하겠습니다."

  3. **실행 (Execute)**
     - 계획에 따라 코드를 수정합니다.
     - **중요**: 리팩토링 과정에서 기존 비즈니스 로직(기능)이 변경되어서는 안 됩니다. 오직 구조만 개선합니다.

  4. **검증 (Verify)**
     - 구문 에러가 없는지 확인합니다.
     - 관련된 테스트 코드가 있다면 실행하여 통과 여부를 확인하라고 사용자에게 제안합니다.
