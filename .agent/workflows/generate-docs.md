name: 기술 문서 생성 (Generate Docs)
description: 선택한 기능이나 모듈에 대한 기술 문서(JSDoc 또는 README)를 생성합니다.

steps:
  1. **코드 스캔 (Scan)**
     - 대상 파일의 코드를 읽고 핵심 로직, 입력 파라미터, 반환 값, 예외 상황을 파악합니다.

  2. **형식 결정 (Format)**
     - **함수/클래스 단위**: JSDoc 형식을 사용하여 코드 바로 위에 주석을 추가합니다. (`@param`, `@returns`, `@throws` 포함)
     - **모듈/기능 단위**: 해당 디렉토리에 `README.md` 파일을 생성하거나 업데이트합니다.

  3. **문서 작성 (Drafting)**
     - 문서는 다음 내용을 포함해야 합니다:
       - **개요**: 이 모듈이 무엇을 하는지 1-2문장으로 요약.
       - **사용법**: 다른 개발자가 복사해서 쓸 수 있는 코드 예시 (Example Usage).
       - **주의사항**: 사용 시 주의해야 할 제약 조건이나 의존성.

  4. **최종 검토 (Review)**
     - 생성된 문서가 실제 코드 동작과 일치하는지 확인합니다. 거짓 정보(Hallucination)가 없어야 합니다.
